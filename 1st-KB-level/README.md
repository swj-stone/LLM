# 第一个LLM搭建的尝试

---

## 缺陷

| 描述 | 后果 |
|------|------|
| 数据问题 | 无多样性、过拟合、无泛化能力、样本过小 |
| 模型架构问题 | RNN/LSTM、无注意力机制、串行计算导致低效 |
| 训练策略问题 | 无学习率调度、无验证集评估、无过拟合预防 |

---

## 结构特征

**模型结构 - 三层神经网络：**

-输入: "在咖啡" (3个字符)
-   ↓
-【嵌入层】: 把每个字变成数字向量 (0.2, -0.5, 0.8, ...)
-   ↓
-【LSTM层】: 理解上下文关系（记得前面说了什么）
-   ↓
-【输出层】: 计算下一个字是什么（143个字的概率）
-   ↓
-输出: "馆" (概率最高的是'馆' 45%，'里' 20%，'的' 10%...)

** 训练过程 - 反复纠正错误**
-循环执行：
-  1. 模型预测 → 猜下一个是"里"
-  2. 正确答案 → 应该是"馆"  
-  3. 计算损失 → 猜错了，惩罚分=2.5
-  4. 反向传播 → 调整模型参数，下次更可能猜"馆"
-重复训练
-损失函数：交叉熵（CrossEntropyLoss）

**生成文本 - 自回归生成**
-输入: "在咖啡"

-第1步: 模型预测 → "馆"，text = "在咖啡馆" + "馆" = "在咖啡馆馆" ❌

-第2步: 用"在咖啡馆馆"继续预测 → "里"，text = "在咖啡馆馆里" ❌❌

-问题：一步错，步步错

-解决方案：温度采样（Temperature Sampling）

-温度=0.1: 永远选概率最高的（保守，容易重复）

-温度=0.8: 偶尔选第二、第三可能的（有创意）

-温度=1.5: 随机选（天马行空，经常乱说）

## 改进方向

**扩大数据集**（最主要）：
- 采用公开真实数据集，而非自己生成
- 尝试至少MB级别的数据
- 数据增强、随机打乱句子顺序、同义词替换

**模型升级**：
- 利用现代框架：Transformer

**训练方式**：
- 数据划分：train/val/test = 0.8/0.1/0.1
- 学习率调度
